# Contributing to Questarr

We welcome contributions to Questarr! Please follow these guidelines to ensure a smooth collaboration.

## Getting Started
1. Fork the repository.
2. Clone your fork: `git clone <your-fork-url>`
3. Install dependencies: `npm install`
4. Create a new branch: `git checkout -b feature/my-feature`

## Development Guidelines
- **Code Style**: We use Prettier and ESLint. Run `npm run lint` and `npm run format` before committing.
- **Testing**: Add tests for new features or bug fixes. Run `npm test` to ensure everything passes.
- **Types**: Use strict TypeScript types. Avoid `any`.
- **Commits**: Use descriptive commit messages.

## Pull Requests
- Use the PR template.
- Ensure all CI checks pass.
- Request a review from a maintainer.

## Reporting Issues
- Check existing issues before creating a new one.
- Provide a clear description of the problem.
- Include steps to reproduce.
